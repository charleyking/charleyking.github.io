shadowsocks搭建完全指南

在ubuntu下安装和配置shadowsocks还是挺简单的。只要按照以下步骤来就好了。

服务器端

  1. sudo apt-get update

  2. sudo python --version （检查python版本）

  3. sudo apt-get install python-gevent python-pip

  4. sudo pip install shadowsocks

  5. touch /etc/shadowsocks.json （创建一个配置文件）

  6. vim /etc/shadowsocks.json （vim编辑配置文件）

#!bin/bash

#filename:shadowsocks.sh
/usr/local/bin/sslocal -s @@@@ -p @@@@ -k @@@@
```

  • 编辑启动文件 sudo vim etc/rc.local

    在exit0上面一行添加nohup bash /home/shadowsocks.sh

  • 保存完成，现在每次开机启动后，shadowsocks服务就会自动启动啦！

浏览器配置

firefox浏览器

  • firefox上的代理插件一般用foxyproxy standard，安装好插件后进入主面板，选择新建一个新的代理。编辑这个代理，在代理详情选项卡中设置host address为127.0.0.1 , socks proxy为socks v5, 密码当然是你之前设置的密码啦。主要是这三个东西，至于其他的大家就可以自由发挥了。

  • 关于这个host address多说一句，为什么这玩意儿会是127.0.0.1而不是我们的服务器地址呢。因为对我们的ss客户端来说，服务端自然是ss的服务端了；然而对于浏览器来说，它的服务端就是ss的客户端，那么服务端地址是本机地址就不难理解了。浏览器的请求信号先发送给ss客户端，再由客户端将加密信号发送给位于国外服务器的ss服务端，达到绕过某墙的目的。

  • 为了达到对特定网站使用代理，而对正常的网站直接连接的目的。还可以订阅一下代理规则。点击主面板中的模式订阅选项卡，添加新的模式订阅，在弹出的面板中订阅地址可以使用http://autoproxy-gfwlist.googlecode.com/svn/trunk/fgwlist.txt，格式选择autoproxy，obfusxation选择base64，其他的自己发挥。

使用wordpress搭建个人博客

花了一天时间搭建了一个自己的网站，准备将其他博客平台上的数据迁移到这里。总的来说，完整的安装过程还是比较繁琐的，还有一些稀奇古怪的问题，对于小白来说简直算是一个大工程了。所以趁现在各种问题还热乎的时候，写上一篇教程，造福群众。嘿嘿嘿。

准备工具

  • putty—可以远程登陆你的主机
  • filezilla—ftp工具，可以将文件上传到你的主机
  • 关于各种工具的用法，善用搜索引擎

购买vps

  • VPS（Virtual Private Server 虚拟专用服务器）技术，将一部服务器分割成多个虚拟专享服务器的优质服务。 每个VPS都可分配独立公网IP地址、独立操作系统、独立超大空间、独立内存、独立CPU资源、独立执行程序和独立系统配置等。vps是一个虚拟服务器,上边没有运行网站的数据库等软件，相当于一个空白的操作系统。需要自行安装各类所需软件。市场上各类vps很多，不过比较好的也就是那么几家。可以看看linux主机推荐，里面的高票回答还是很靠谱的。我本人现在使用的是Ubiquity hosting,因为现在有优惠大酬宾，新用户注册可以获得10美元的账户金额，另外使用25BUCKS这个优惠码可以再获得25美元账户金额。如果购买5美元一个月的vps套餐，就可以免费使用7个月了，蛤蛤。不过购买这个vps需要先用信用卡或paypal往账户中充入10美元（也就是60RMB用上9个月）,还是相当实惠的。

  • 购买过程中，配置vps的时候记得给vps安装上ubuntu系统，因为本文使用的例子就是ubutu系统。 购买成功后，你会收到确认邮件，把主机的ip地址和登录密码记好。

购买域名

  • 域名就是你的网址，虽然以后直接用ip地址也能进入网站，不过你总不希望每次在地址栏输入一串数字吧。

  • 购买域名的地方也是很多，可以自行搜索一下。我使用的是namecheap的服务，这家的域名就和它的名字一样，真心cheap。我的域名charleyking.club只要六块钱一年。如果英文不好在万网买也是一样的。

  • 购买成功后，在个人面板中将域名解析到vps的ip。

ubuntu下软件安装方法

ubuntu下安装软件有三种方式，分别为在线安装apt-get方式，软件商店安装方式和

1. 软件商店安装方式

  • 这种方式对经常使用windows系统的同学来说最为简单，因为其界面是图形化的。现在的ubuntu系统默认就有软件商店，所以只要在里面搜索一下需要的软件，点击安装就可以。唯一的缺点就是很多不太知名或者刚出来的软件搜索不到，在这种情况下。我们只能从这些软件的官网上下载对应的安装文件或者压缩包，然后自己手动安装了。

2. 在线安装apt-get方式

  • 这种方法是最方便的了，因为只需要敲一行命令就好了。

  • sudo apt-get install packagename

  • 关于apt-get的其他方法，该命令一般需要root权限，所以前面要添加sudo命令

  1. apt-cache search package 搜索包
  2. apt-cache show package 获取包的相关信息
  3. apt-get install package 安装包
  4. apt-get -f install 修复安装
  5. apt-get remove package 删除包
  6. apt-get update 更新源
  7. apt-get upgrade 更新已安装的包
  8. apt-get dist-upgrade 升级系统
  9. apt-cache depends package 了解使用依赖
  10. apt-cache rdepends package 是查看该包被哪些包依赖
  11. apt-get source package 下载该包的源代码
  12. apt-get clean && sudo apt-get autoclean 清理无用的包
  13. apt-get check 检查是否有损坏的依赖

3. 手动安装方式

  • “deb格式文件”：这种文件格式和windows下的.exe格式是一样的，是ubuntu系统下的可执行安装文件，直接双击即可执行安装程序。另外也可以使用dpkg -i filename.deb命令来安装。

  • “.rmp格式文件”:这种文件格式可以使用rmp -i filename.rmp命令来安装，不过第一次要先使用sudo apt-get install rmp命令来安装rmp包。

  • “.tar.gz/.tar.bz2格式文件”:这种格式文件其实是软件源码的压缩包。如果是.tar.gz格式我们先使用tar -zxvf filename.tar.gz命令来解压缩,进入解压后的文件夹目录，再通过./configure命令来配置编译环境，然后通过make命令来进行编译，最后通过make install命令来进行安装。如果是.tar.bz2格式，需要改变的就是第一步的参数，将-zxvf变成-jxvf就好了

windows下硬盘安装ubuntu双系统

对于很多想要体验linux系统但却不舍得完全舍弃windows的同学来说，安装双系统可能是最好的选择了。这里我将介绍在win7下安装ubuntu的方法，很简单哦。

首先需要的准备的是两个东西，一个是EasyBCD软件，http://yunpan.cn/cdEjEZqsPLnqi （提取码：fe30），另外一个自然是ubutu的镜像文件了，这是官方的下载地址http://www.ubuntu.com/index_roadshow。

第一步：分区

在计算机上右键–管理–磁盘管理

装Ubuntu分配的硬盘大小最好是20G以上（不要太小）。分配好空间后，在磁盘管理中右击相应的分区，选择删除卷。安装好的ubuntu的分区，在Windows下是看不到的，但是进入Ubuntu是可以访问Windows的磁盘的。这才是真正的双系统。而采用wubi安装的话，就相当于在windows下安装一个软件了。

第二步：使用EasyBCD软件

打开软件后，按照以下步骤操作：

点击“添加新条目”

点击NeoGrub选项卡

点击“安装”

点击“配置”

出现一个menu.lst文件，是ubutu的引导文件，你需要将以下内容替换进去

注意将粗体部分改成自己下载的镜像名称。保存。

第三步：操作镜像文件

将准备好的ubutu镜像解压缩，找到casper文件夹，把里面的initrd.lz和vmlinuz解压到C盘，把.disk文件夹也解压到C盘，然后在把整个iso文件复制到C盘。最新的64位系统解压出来是vmlinuz.lcr，注意将扩展名删除，否则重启会直接进入windows系统。

第四步：重启安装

哈哈，到了激动人心的安装过程了，不要心急。按照提示一步一步来，非常简单。

重启之后你会发现有两个启动菜单，而且有一个是很眼熟的NeoGrub，选择这个选项。接下来就是进入ubutu的安装桌面了，这里会弹出一个所有键盘快捷键的图，直接关闭就好了，快捷键以后慢慢学，现在不着急。

按Ctrl+Alt+T 打开终端，输入代码:sudo umount -l /isodevice这一命令取消掉对光盘所在 驱动器的挂载（注意，这里的-l是L的小写，-l 与 /isodevice 有一个空格。），否则分区界面找不到分区。记得输入两次这个命令，已确保取消挂载。

点击桌面上的“安装ubuntu”，开始接下来的七步配置。

第五步：七步配置

完成这七步，就可以使用令人期待的ubuntu系统了，让我们一步一步来。

  • 欢迎菜单：选择系统语言，当然简体中文了，如果你英语够好想要装逼就下拉至“English”。下一步

  • 无线连接菜单：并不需要网络，直接选择“我现在不想连接wifi”。下一步

  • 准备安装ubuntu：这一步并不需要任何操作，直接下一步

  • 安装选项：选择最下方的“其他选项”，准备为ubuntu分区操作。下一步

  • 安装类型菜单：注意，这是最重要的一步操作。选择为ubuntu准备的那个空白分区，点击下方的加号，弹出一个菜单。调整大小，将这个分区大小减小2GB（2048MB）作为swap分区。“用于”这个选项选择“Ext4日志文件系统”。“挂载点”选择“/”。确定后我们发现原来的ubuntu分区下面出现了一个2GB的小分区，就是刚才分出来的，选择这个分区，点击下方的加号，在“用于”选项中选择“swap”，确定。OK，最下面的启动项不用改。下一步。

  • 地点菜单：你在什么地方？地点直接按照默认的shanghai就好了。下一步。

  • 自动安装，等待系统配置完成

哈哈，搞定了，简单吧？安装完成后，在弹出的菜单选择“立即重启”。重启之后，尽情享受ubuntu系统带给你的快感吧！

windows 快捷操作高级篇

上次介绍了windows系统上自带的一些快捷键，虽然有些很方便，但是毕竟还是太少了，而且无法自定义专属于自己的快捷键。所以我写了这么一篇教程，主要介绍windows平台上的autohotkey和chrome浏览器上的vimium插件。当掌握这两个神器之后，你会发现自己的工作效率会得到惊人的提升。

玩游戏的同学应该都听说过按键精灵，它可以通过程序自动实行按键指令。autohotkey相比于按键精灵更加强大，几乎可以这么说–windows平台上所有可以自动化的东西都能够用autohotkey来实现。所以，经常有人把它称作是windows顶级神器。这神器唯一的缺点或许就是太强大了，导致刚上手的时候一脸懵逼。当然一般用户也用不到各种高级功能，只需要实现以下几个简单的功能，你就不会后悔安装autohotkey了。

  • 字符串自动补全

    很多时候我们经常要重复输入一些句子，比如手机号码，账号，邮箱或者密码什么的，这时候使用ahk的字符串补全功能能够帮助你节省很多时间。当你输入关键词后，自动出现和你的关键词对应的字符串。

  • 快捷键打开程序

    windows自身就有快捷键可以打开一些软件，比如用win+num可以调用任务栏上相应位置的程序。但是如何更方便更灵活地使用快捷键呢？

    其实，你只要在ahk的脚本编辑器里输入类似这样的命令，就可以实现这个目标了。

    !e:: 

    run C:\Windows\System32\ev

    return

  • 快捷键打开网站

    什么？网站也可以？那是当然的啊，网站本质上也是一个小程序嘛。所以只要把打开程序的命令中程序位置换成网址就好了。

  • 替换组合键

    使用笔记本电脑的同学可能经常有这样的感受，调用alt + f4来关闭一个程序的时候会很别扭，因为这两个键位置相差有点大，尤其是有些笔记本电脑使用功能键还必须先按下Fn，于是就不得不同时按住三个键。这就有点尴尬了，然而，使用ahk能够轻松解决这个问题。只需要将你顺手的快捷键设置为调用alt+Fn+F4这个组合键就可以了。

更多的功能就不展开来了，有兴趣的话可以到官网和中文论坛看看，有很多有趣的帖子。

windows 快捷操作进阶篇

上次介绍了windows系统上一些自带的常用快捷键，有些确实很方便，也满足了我们的一部分需求。但是我们追求效率的步伐怎会止步于此？这一次我将会进一步介绍windows上提升效率的方法。

一：运行

打开运行对话框的方法在上一篇提到过，就是win+r。这个运行框虽然可以打开文件，文档，程序等，但是打开文件和文档并不方便，我在后面会提到快速打开文件的更方便的方法。

win+r用来打开程序是非常方便的，但是需要一些配置，配置完成后，只要在对话框中输入你为程序预先设置好的名字就能快速打开该程序。比如我为chrome浏览器设置的名字为c，因此win+r之后，只需键入一个c，就可以快速打开浏览器。是不是比返回桌面去点击浏览器的快捷方式方便多了？

下面是配置方法：

找到你想要设置快捷键的程序文件（在你安装程序的文件夹里，后缀.exe），右键创建快捷方式，然后将这个快捷方式的名字改成你最喜欢的，越简单越好（当然前提是容易记住），接着将这个快捷方式移动到C:windowssystem32文件夹里面。

现在试一试win+r，输入设好的名字，相应的程序马上就打开了。

ps：其实直接按win键调处开始菜单也可以直接输入简易名称打开相应程序的，比按win+r还更加方便一些，至于喜欢哪个方法就看个人喜好了。

三：rainmeter

windows 快捷操作基础篇

从网上汇总了一些windows系统上常用的快捷键，结合自己的使用经验，将平时使用电脑时最常用的快捷键记录在此。注意这里罗列的是平时最常用到的，其他的一般来说不怎么用到的并不在此列，如果想要完整列表，请参考该链接。键盘快捷键

  • Enter 对于许多选定命令代替单击鼠标
  • windows键或CTRL+ESC 打开开始菜单
  • CTRL+SHIFT+ESC 或 CTRL+ALT+DELETE打开任务管理器
  • DELETE删除被选中的项目，如果是文件，将被放入回收站
  • SHIFT+DELETE 删除被选中的项目，如果是文件，将被直接删除而不是放入回收站
  • CTRL+SHIFT+N 新建一个新的文件夹
  • CTRL+A 全选文件夹内的文件
  • CTRL+O 打开“打开文件”对话框
  • CTRL+P 打开“打印”对话框
  • CTRL+S 保存当前操作的文件
  • CTRL+W关闭当前的窗口
  • CTRL+ X剪切被选择的项目到剪贴板
  • CTRL+C 复制被选择的项目到剪贴板
  • CTRL+V 粘贴剪贴板中的内容到当前位置
  • CTRL+Shift/space 切换中英文输入法
  • CTRL+F4 关闭当前应用程序中的当前文本

sql syntax

学了一个月的java，开始有入门的感觉。这段时间接触到了java的JDBC, 发现学习这部分的内容还是要有SQL的基础，于是花费了几天时间学习了一下SQL语法，并将其总结于下。

操作数据库的方法有很多，而使用SQL是最直接，最普遍的操作方法。SQL就相当于数据库提供给外界的数据操作接口，无论是手动还是编程的方式，都可以向数据库发送SQL命令，数据库会根据命令做出相应的响应。

SQL的语法相当简单，但是功能却不可小觑，它能对数据库实现完全的操控。而这种操控主要体现在四个方面：增，删，查，改。

首先是查找，这是操作数据库的基础，关键字SELECT

  • SELECT FROM WHERE ORDER BY ; //最基本，熟练掌握
  • 高级过滤 SELECT [a,b] FROM table WHERE condition1 AND/OR condition2;
  • 通配符过滤 SELECT [a,b] FROM table WHERE a LIKE ‘fish%’; // [%],[_],[[]]三个通配符
  • 计算字段 SELECT a + ‘(‘+b+’)’ FROM table ORDER BY a; //拼接
  • SELECT RTRIM(a)+’(‘RTRIM(b)’)’ AS c FROM table ORDER BY a; //列别名
  • 函数字段 SELECT [a,b] FROM table WHERE SOUNDEX(a) = SOUNDEX(‘tim’); //根据发音匹配
  • 汇总数据 SELECT AVG(a) AS c FROM table WHERE condition; // COUNT(), MAX(), MIN(), SUM()汇总函数
  • 分组数据 SELECT a, COUNT(*) AS c FROM table GROUP BY a; // 创建分组
  • SELECT a, COUNT(*) AS c FROM table GROUP BY a HAVING condition; // 过滤分组
  • SELECT - FROM - WHERE - GROUP BY - HAVING - ORDER BY // 分组语句的关键词顺序
  • 组合查询 SELECT [a,b] FROM table WHERE c IN(‘MI’) UNION SELECT [a,b] FROM table WHERE condition;

插入，关键字INSERT

  • INSERT INTO (col1,col2) VALUES(val1, val2);
  • INSERT INTO (col1, col2) SELECT col1, col2 FROM ; // 复制表数据

更新，关键字UPDATE

  • 更新数据 UPDATE SET WHERE ;

删除，关键字DELETE

  • 删除数据 DELETE FROM WHERE ;
  • 创建表 CREATE TABLE table {…};
  • 更新表 ALTER TABLE table ADD a CHAR(20);
  • 删除表 DROP TABLE table;

start using github

一直以来，github的大名就如雷贯耳。虽然我半年多前就从了解到了这个神奇的网站，而且趁着当时的一时兴趣注册了账户，但是对于那时候的我来说这个网站还是太复杂了点，毕竟半年前的我还没有开始写代码，所以就算注册了这个代码管理软件也丝毫没有用处–总不能真的拿来当作网盘吧。

于是眼睛一闭一睁时间就到了11月，半年来由于经常混迹于各种技术论坛，github的字眼是从来没有少出现在我眼前。我也不止一次兴致勃勃地打开那个熟悉的网页，鼓捣一番，然后又默默关掉–真的是用不来。每当新建了库之后，我根本不知道该如何上传，所以到后来我的主页上看起来虽然有好多的库，然而点进去之后却都是空的……
这种情况终于在前天开始改变了，想来自己学了大半年的cs，总还是有一些产出的，我也很想把自己的这些成果给别人看啊。在这种强烈的内因作用下，我下定了决心这一次要彻底搞明白github并将它使用起来。
查阅了资料之后发现原来基础是先学会git的使用，于是按照大家的推荐看了progit这本书，确实很好，看完前三章也就对git有了比较全面的了解并且基本上能够上手了。然后结合网上的各种教程，下载了git软件，开始对着教程在命令行里敲。失败了几次之后（第一次是生成密钥是密码弄错了导致后来根本上传不上去，还有一次是在github上删除了一个文件然后本地没有fetch下来导致又上传不上去…..总之踩了很多坑），终于成功把一个本地的库上传到了github上。成功了一次之后，那就是一发不可收拾了，我又使劲地传了四五个库上去……每次看到命令行出现的成功提示，真的是成就感满满啊。
