java socket

什么是socket？

socket通常也称作”套接字”，用于描述IP地址和端口。应用程序通常通过socket向网络发出请求或者应答网络请求。客户端和服务器端各有一个socket，我们可以将其简单理解为一扇门，各种数据从这扇门里通过。假若有一个数据包要从客户端发送到服务器，那么这个数据包会从客户端的门里出来，经过网络链路，到达服务器门口，通过这扇门到达服务器内部，再处理数据包中的数据。

常用API

java.net.Socket继承于java.lang.Object，有八个构造器，但是方法并不多，常用的有如下几个：

  • Accept()方法可以产生”阻塞”，使程序停留此处，直到接受到一个连接，并且返回一个客户端的Socket对象实例，通常”阻塞”放在死循环中。

  • getInputStream()方法获得网络连接输入，同时返回一个IutputStream对象实例。

  • getOutputStream()方法连接的另一端将得到输入，同时返回一个OutputStream对象实例。

java网络编程

网络编程最常见的是client-server模型，过程可以概括为：

  • One program, called the server blocks waiting for a client to connect to it
  • A client connects
  • The server and the client exchange information until they’re done
  • The client and the server both close their connection

服务器端

服务器端的ServerSocket监听端口，直到接收到一个Socket,再通过输入流取得socket的内容，并对其进行操作。最后记得关闭流和socket。

这个服务器程序只能接收一个客户端连接，如果想要接收多个连接怎么办？很简单，只需要在accept到一个socket之后，新建一个线程，在新线程中进行io操作就可以了。
