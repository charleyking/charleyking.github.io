javaEE 三大框架介绍

Struts

使用Struts开发Web应用程序可以分为4个步骤：

  1. 搭建Struts开发环境，包括必须的jar文件，web.xml配置和创建struts-config.xml文件
  2. 实现Views，主要是创建JSP文件
  3. 实现Model，主要是创建和配置ActionForm类
  4. 实现Controller，主要是创建和配置Action类

Spring

它是一个基于IOC和AOP的构架多层j2ee系统的框架，但它不强迫你必须在每一层中必须使用Spring，因为它模块化的很好，允许你根据自己的需要选择使用它的某一个模块；它实现了很优雅的MVC，对不同的数据访问技术提供了统一的接口，采用IOC使得可以很容易的实现bean的装配，提供了简洁的AOP并据此实现Transaction Management

Hibernate

Hibernate是数据持久化的一种解决方案，同时也是对象关系映射模型(ORM)的一个典型代表。所谓的ORM，可以简单理解为java面向对象的模型和数据库的关系模型之间的桥梁。它向上隐藏了关系模型，使得开发者能够按照面向对象的思想来操作关系数据库的数据。

在web程序中整合Struts, Spring, Hibernate

采用SSH组合开发web程序是java web开发领域的经典模式，在一个web应用程序中，三者各司其职。

  • Struts：表现层，主要作用是接收请求和展现请求结果
  • Spring：业务层，将Struts和Hibernate穿插在一起
  • Hibernate: 数据层，提供对数据进行相关操作的接口和实现

整合三者有以下步骤：

  1. 完成Struts配置，包括web.xml和struts-config.xml
  2. 完成Hibernate配置文件hibernate.cfg.xml
  3. 在web.xml为Spring配置应用程序上下文监听器
  4. 用Spring提供的controller替换Struts原有的controller
  5. 用Spring提供的SessionFactoryBean来读取hibernate.cfg.xml，完成Spring对Hibernate的整合
  6. 保存必要的jar文件到”WEB-INF/lib”文件夹中

设计模式

聊聊设计模式

MVC设计模式(Model-View-Controller)

MVC设计模式的主要思想是把控制器，数据模型和视图分离，达到高内聚和低耦合的作用。

以java web开发为例，有两种普遍的开发模式。模式一使用JSP+JavaBean技术将页面显示和业务逻辑分离，JSP显示页面，JavaBean处理数据和逻辑。模式二就是把处理逻辑的代码提取出来成为一个独立的角色，这个角色就是控制器，而存取数据的JavaBean就是数据模型，此时模型二就符合了MVC的设计模式。

简而言之，不适用框架的情况下，可以使用以下步骤来实现MVC的设计模式：

  1. 开发Model：主要是开发JavaBean，它的作用是数据的存取
  2. 开发Views：主要是开发JSP，它的作用是从JavaBean中读取数据进行展示
  3. 开发Controller：主要是开发Servlet，它的作用是根据请求的参数或者URL不同，转发相应的请求，一般还会进行数据的创建和填充

使用MVC设计模式的代表例子有Struts框架。

算法大全

排序算法

选择排序

选择法排序的思路：

把第一位和其他所有的进行比较，只要比第一位小的，就换到第一个位置来，比较完后，第一位就是最小的
然后再从第二位和剩余的其他所有进行比较，只要比第二位小，就换到第二个位置来，比较完后，第二位就是第二小的

以此类推

冒泡算法

冒泡算法基本概念是：依次比较相邻的两个数，然后将小数放在前面，大数放在后面。相当于气泡逐渐往上升，所以形象的称为冒泡排序。

插入排序

插入排序类似于打扑克牌，每抓一张牌都要把它按照一定顺序插入到手中已有的牌中。

快速排序

快速排序是排序算法中效率最高的一种。它利用递归原理，把数组无限制分割为两个部分，直到所有数据都排好位置。排序算法是对冒泡算法的一种改进。

jdbc

我们来聊一聊JDBC，它是专门用来执行SQL的java API，为java程序员访问数据库而准备。各种数据库差异巨大，而JDBC隐藏了细节，提供了一种统一的方法来访问操作数据库。
JDBC由两部分构成，第一部分是供程序员调动的API，另一部分是需要数据库厂商实现的SPI，也叫做驱动程序。JDBC提供了一系列java接口给数据库厂商，让他们去实现这些接口。然后提供另一组通用接口给程序开发人员，通过这种方式实现了通过一组API来操作不同数据库。

JDBC操作数据库步骤

  1. Class.forName(“class name of driver”); // register the database driver
  2. Coonnection conn = DriverManager.getConnection(“URL”, “usrename”, “password”); // get a database connection
  3. Statement stmt = conn.createStatement(); // create a new statement
  4. stmt.executeUpdate(“sql cmd of INSERT/UPDATE/DELETE”); // execute sql
  5. ResultSet rs = stmt.executeQuery(“sql cmd of SELECT”); // get a result set
  6. while (rs.next()) { do something to set } // do operation to set
  7. rs.close(); stmt.close(); conn.close(); // close the connection

可以看出，连接数据库的操作步骤是比较固定的。其中第六步对结果集的处理是可选的，另外第七步关闭数据库连接很重要，是一个良好的编程习惯。

JDBC 事务

假设一个银行转账的程序，在某一次转账中，出现了一些问题，造成一方已经扣款，而另一方并没有存入相应的金额。这样的错误是毁灭性的，为了避免这样的错误发生，我们需要使用事务。事务其实并不复杂，而且JDBC对它提供了充分的支持。

一个事务的结束只能有两种形式，提交和回滚。操作成功则提交，产生永久性的修改。操作不成功则回滚，恢复到事务开始前的状态。只有如此才能从根本上避免上述错误的发生。

那么JDBC中如何使用事务呢？默认情况下，JCBC自动提交事务，也就是说，执行SQL的语句完成后，就立刻提交事务。但是多数情况下，我们不希望马上提交，而是自主控制，这样在发生了异常或者参数出现问题后，能够回滚事务。以下是一个典型的使用JDBC事务过程。

使用gitignore忽略文件

gitignore介绍

在工程实现过程中，会生成一些中间文件，或者在项目中的部分文件是不需要进行版本管理的。如何忽略这些文件呢？Github提供这种功能，可以自己指定哪些文件可以不被管理。具体方法是在版本管理的根目录下（与.Git文件夹同级）创建一个.gitignore文件，在.gitignore中具体指出不需要进行版本管理的文件就可以了。

文件.gitignore的格式规范：

  • #为注释
  • 可以使用shell所使用的正则表达式来进行模式匹配
  • 匹配模式最后跟”/“说明要忽略的是目录
  • 使用！取反（例如目录中包含test.a，并且gitignore文件中包含 *.[oa]，如果在文件中加入 ！test.a 表明忽略除test.a文件以外的后缀名为.a或者.o的文件）

语法

  • 以“/”为开头表示目录
  • 以“？”为开头通配单个字符
  • 以“*”开头通配多个字符
  • 以“[]”表示包含单个字符的匹配列表

举例

*.[oa] // 忽略所有以O或者a为后缀的文件

.gitignore // 忽略gitignore文件本身

main // 显式指定，忽略名称为main的文件

.metadata/* // 忽略所有.metadata文件夹的内容

/.metadata/* // 忽略根目录下.metadata文件夹的内容

注意

  • gitignore的匹配按照从上到下的顺序，也就是说前面的语句优先级更高

配置完gitignore文件后，执行git status命令，发现gitignore中的文件都不再被跟踪，也就实现了我们的目的。

java socket

什么是socket？

socket通常也称作”套接字”，用于描述IP地址和端口。应用程序通常通过socket向网络发出请求或者应答网络请求。客户端和服务器端各有一个socket，我们可以将其简单理解为一扇门，各种数据从这扇门里通过。假若有一个数据包要从客户端发送到服务器，那么这个数据包会从客户端的门里出来，经过网络链路，到达服务器门口，通过这扇门到达服务器内部，再处理数据包中的数据。

常用API

java.net.Socket继承于java.lang.Object，有八个构造器，但是方法并不多，常用的有如下几个：

  • Accept()方法可以产生”阻塞”，使程序停留此处，直到接受到一个连接，并且返回一个客户端的Socket对象实例，通常”阻塞”放在死循环中。

  • getInputStream()方法获得网络连接输入，同时返回一个IutputStream对象实例。

  • getOutputStream()方法连接的另一端将得到输入，同时返回一个OutputStream对象实例。

java网络编程

网络编程最常见的是client-server模型，过程可以概括为：

  • One program, called the server blocks waiting for a client to connect to it
  • A client connects
  • The server and the client exchange information until they’re done
  • The client and the server both close their connection

linux学习指南

为了系统地学习一下linux相关知识，我花了三天的时间看完了那本经典的《鸟哥的linux私房菜》。所谓的经典必然是有其过人之处，而看完这本书之后，我毫无疑问地加入了强烈推荐这本书的行列。

其实，要全面了解linux系统，你必须要学习（至少了解）以下这些内容：

  1. 计算机硬件系统：其中最主要了解的是硬盘，毕竟要安装linux需要对硬盘进行分区等各种操作。

  2. 文件系统：文件系统是文件在硬盘上的存储方式。

  3. 目录树：linux系统下所有文件均由根目录衍生出来，因此形成树状结构，这和windows系统是一个明显的区别。关于文件系统是如何挂载到目录树上的，这也是一个不容忽视的概念。

  4. 终端/壳程序：很多人搞不清这个terminal和shell究竟有什么区别。这两者确实比较容易混淆，前者是在全文本模式下的界面，而后者是在窗口模式下调出来的一个程序，和windows系统下的cmd程序类似。进入terminal可以按Ctrl+Alt+F1~6,总共有6个terminal可以使用。而按Ctrl+Alt+F7可以进入窗口模式，然后按Ctrl+Alt+t调出shell程序。

  5. 命令：使用linux自然要知道命令啦，虽然说桌面使用的话其实不怎么用得到命令，但是几个常用的命令还是要了解一下的。

  6. shell script：和windows下的批处理程序.bat类似，主要用途是一次性执行多命令。

  7. 权限管理：linux系统最重要的就是权限管理，这也是它区别于其他系统的一大特色。由于它是一个多人多任务的系统，且主要用于服务器端，各种权限的管理就是系统安全性的保证，所以学习linux一定要把握好权限这一关。

  8. vim：linux下最重要的编辑器自然是vim了，虽然大家都把这玩意儿看的很难，但是多上手使用几次之后发现其实也就是多了几个命令而已。至于为什么一定要学好vim呢？最主要的原因是很多软件或者程序会默认调用vi，而vim 又是vi的升级版。不会这玩意儿，到时候会多很多麻烦。

  9. 正则表达式：嘿嘿，想必大家都早有耳闻了，大名鼎鼎的regular express。这玩意儿也是一些特殊符号多了些，本身并不难，多练练多记记就好。

  10. 帐号管理：多人多任务嘛，不同的人自然有不同的帐号了，那么这个帐号的管理也是相当重要了。这里的帐号管理应当与权限管理结合起来。

  11. 例行任务：就是计划任务啦，分为一次性任务和周期性任务两种。

  12. 程序管理

  13. 系统服务

  14. 登录档：要维护linux系统，必须学会查看登录档。因为登录档里面记录了系统运行的各种日志文档，当系统由于各种原因出现故障的时候，就需要自己打开登录档查看上次出现问题的原因了。

  15. 开机流程：详细地了解开机流程是非常有必要的，能够加深对系统运作的了解。

  16. 软件安装：对于桌面系统用户来说，软件安装才是最主要的问题。

  17. 核心编译：所有的发行版的核心都是一样的，在什么情况下需要自己编译linux核心呢。当你需要在嵌入式设备中安装linux系统时，由于大小的限制，就需要自行对核心进行精简。

  • 我个人认为，要深入了解linux，自行查看系统里的各种文档是最重要的，没事就打开这些文档看看里面提供的各种功能，说明，帮助。这些文档简直就是最好的教科书。

安装使用ubuntu问题汇总

很早以前就安装了ubuntu系统，可是一直没怎么用，也没有深入研究。这两天重装了一下windows，顺带着也重新装了一遍最新的ubuntu。期间碰到了不少问题，一个个解决也花费了不少时间。所以把这些问题都汇总在这里，以后再碰到方便查询，以后在使用ubuntu的过程中遇到的更多问题和解决办法，我也会添加在这里。

一：安装windows和ubuntu双系统

参考我的这篇文章win7下硬盘安装ubuntu双系统

二：ubuntu系统鼠标闪烁问题

很多人安装好ubuntu后使用时会发现鼠标会不停闪烁，感觉异常难受。其实这不是系统安装出了问题，只是没有设置好。解决办法很简单，打开系统设置，点击“显示”，你会发现显示面板上方有一个内置显示器和一个未知显示器。其实就是这个未知显示器在捣乱，我们只要将其关闭就好了。点击“未知显示器”，在下方的选项中将其关闭。OK，问题解决。

三：ubuntu网络连接问题

在网络连接里面有很多设置项，分为“有线”，“无线”，“移动宽带”，“vpn” ，“DSL”。注意家里的宽带是在DSL里输入账号密码，不要选择“有线”或“移动宽带”。

四：grub里面不能选择windows进入

切换双系统的时候，在grub菜单选择windows，发现过几秒钟直接返回grub选择菜单，无法进入windows。这个比较郁闷了，我回想了一下问题可能出现在哪里，随后想起来在安装ubuntu分区这一步时，bootloader这个选项我不是用默认设置的而是将其改成了windows的sda1那个分区，问题应当出在这里。在网上查找了一番，果然如此。解决办法是很简单，背后的原理稍微有点费脑。参考这个链接安装Ubuntu后Windows 7无法启动问题分析
